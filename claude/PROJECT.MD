# PROJECT.MD тАФ Triangular Number Matrix Calculator

> Last updated: 2026-03-21 (session 2) | Current phase: Phase 2 in progress тАФ pyramid display algorithm unresolved

---

## What This Project Is

A **POC тЖТ Production** calculator demonstrating a novel mathematical framework for computing triangular numbers of astronomically large **repdigit numbers** (e.g. 3333тАж3333 with 10 billion digits) in near-constant time тАФ without using classic `T(n) = n(n+1)/2` arithmetic division, which breaks down at massive scale.

The core insight: triangular numbers of repdigit numbers follow deterministic, repeating **vertical constants** and **horizontal patterns**. The calculator *assembles* the answer from those patterns rather than computing it.

---

## Mathematical Core

### Repdigit Numbers
A number made entirely of one repeated digit: `2`, `22`, `222`, `2222`, etc.

### Pattern Assembly
For any repdigit `d` of length `L`, its triangular number takes the form:
```
[lc prefix?] + [hpl tiled] + [vpc constant] + [hpr tiled]
```

- **vpc1тАУvpc9** тАФ 3тАУ4 char vertical pattern constant. Which one to use: `vpc{digital_root(d ├Ч L)}`
- **hpl** тАФ horizontal pattern left (9-char tile, repeated left of vpc)
- **hpr** тАФ horizontal pattern right (9-char tile, repeated right of vpc)
- **lc** тАФ optional 1-char left prefix (digits 5, 7, 9 only)
- **rpr** тАФ flag for digit 1: right pad is 1 char shorter than left

### Digital Root
`DR(n) = 1 + (n-1) % 9` (or 9 if `n % 9 == 0`). Used to select vpc. Computed by `_REPDIGIT_CACHE.getDigitalRoot(digit, length)`.

### Special Digits
- `3`, `6`, `9` тАФ all 9 vpcs are identical (`561`, `2211`, `950`); simpler assembly
- `1`, `2`, `4`, `5`, `7`, `8` тАФ 9 distinct vpcs + distinct hpl/hpr
- Digit `1` has `rpr=Y` тАФ one fewer character on right side

### Non-repdigit Proximity
`increment_by_gmpy_count(num1, result, offset)` тАФ jump to nearest repdigit T(n), then add/subtract `offset` via gmpy2 counting. Phase 4 item.

## Calculator Operations

| Form `operation` value | Core method | Notes |
|------------------------|-------------|-------|
| `tri_matrix` | `repDigitTriangularNumber()` | In-memory string assembly. Returns `str`. |
| `tri_matrix_stream` | `repDigitTriangularNumberStream()` | Chunk-writes to disk. Use `collect_result=True` for web UI. Returns dict. |
| `tri_matrix_memory` | `repDigitTriangularNumberMemory()` | Generator-based chunks. Use `collect_result=True` for web UI. Returns dict. |
| `tri_matrix_random` | `repDigitTriangularNumberRandomAccess()` | Random window access from ShortFormBigNumber. Returns dict. |
| `tri_div_gmpy2_formula` | `triangular_number_gmpy2_division()` | Classic n(n+1)/2 via gmpy2. Ground truth baseline. |
| `tri_div_simpy_formula` | `triangular_number_sympi_division()` | Classic formula via sympy. Slower than gmpy2. |
| `add/subtract/multiply/divide` | `ManualBigNumber.*` | Manual big-number arithmetic helpers. |

**Critical rule**: For web UI, all dict-returning modes (`tri_matrix_stream`, `tri_matrix_memory`, `tri_matrix_random`) must use `collect_result=True`. With `collect_result=False`, the TN string is never built in memory тАФ result stays a stats dict, and `str(dict)` gets displayed instead of the triangular number.

## Key Classes

### `TriangulaNumberMatrix` (`core/calculator.py`)
Main calculator. `TriangulaNumberMatrix("1")` тАФ constructor arg is legacy/unused. Holds `self.matrix` dict (all vpc/hpl/hpr/lc/rpr for digits 1тАУ9). All 4 calculation modes live here.

### `ManualBigNumber` (`core/calculator.py`)
String-based big number wrapper. Arithmetic methods + fallback TN methods (gmpy2, sympy).

### `ShortFormBigNumber` (`core/calculator.py`)
Frozen dataclass: `b` (base), `l` (length), `n` (digit). Parsed via `ShortFormBigNumber.from_ops(ops_string)`. Used by `tri_matrix_random`.

### `RepdigitCache` (`core/repdigit_cache.py`)
Precomputed block-sum cache for digits 1тАУ9 up to 10 billion. Powers constant-time digital root lookups. Module-level singleton `_REPDIGIT_CACHE` тАФ **never reinstantiate per request**.

### File Input Mode (important wiring detail)
The File Input Mode radio is **outside the calculator form** (bottom of page). Two hidden fields inside the form sync it:
- `#file-name-hidden` (name=`file_names`) тАФ set by JS on disk selection
- `#file-mode-hidden` (name=`file_mode`) тАФ synced from radio via inline `<script>` on page

Flask reads `file_mode` to decide whether to read from disk or use the `num1` textarea value.

## Result Display Rules

- Flask computes `result_str`, `result_total_chars = len(result_str)`, `result_preview = result_str[:10000]`
- Template receives `result` (max 10K chars) and `result_total_chars` (actual full count)
- `<pre id="result-full" style="display:none;">{{ result }}</pre>` тАФ JS reads this for nav
- Shows amber warning when `result_total_chars > 10000`
- Nav window: 200 chars visible, keyboard shortcuts Alt+тЖР/тЖТ

## Critical Rules (learned from bugs)

1. **`.strip()` on file reads** тАФ `read_file_content()` must `.strip()` the result. A trailing newline increases `len(number_str)` by 1, shifts DR by 1, selects wrong vpc. Produces wrong TN silently.

2. **`file_load_time = "0"` not `0`** тАФ used in string concatenation for stat file writes. Must be `str`.

3. **`collect_result=True` for web** тАФ all dict-returning calculator modes must use this flag. Without it, `result` is a stats-only dict тАФ `str(dict)` gets displayed instead of the TN.

4. **`if file_mode == "disk" and file_name`** тАФ guard must check both conditions. `if file_name` alone reads from disk even in HTTP mode if the hidden field has a stale value from a previous disk selection.

5. **`_REPDIGIT_CACHE` is a module-level singleton** тАФ never reinstantiate per request.

6. **File write paths need `os.makedirs(..., exist_ok=True)`** before writing stat/output files.
